Javascript

Data Structures

Numbers

- Includes integers, +infinity and -infinity, and NaN (not a number)
- Numbers can be a bit weird in javascript because not a lot of digits are stored in memory. While
0.1 + 0.2 == 0.30000000000000004 returns true,
0.1 + 0.2 == 0.3000000000000004 (one less zero) returns false.

There's a built-in object called Math that helps out with a few mathematical functions and constants.
Math.sin(0) // 0
Math.round(Math.exp(2)) // 7
Math.pi // 3.141592653589793

There are built in functions that can, for example, convert characters to integers (the 10 = the base).
parseInt("123", 10) // 123
parseInt("ummm") // NaN
+"10" // 10

Strings

substr(begin, length)
'hello world!'.substr(6, 5) // "world"
"hello".charAt(0) // "h"
"Eww, ice cream".replace("Eww,", "I like") // "I like ice cream"
"hello".toUpperCase() // "HELLO"
"ice" + " " + "cream" // "ice cream"
Note that when coercing numbers to characters, order matters. If the numbers come first, they'll be added together before being coerced to characters. "1" + 2 + 3 // "123"
2 + 3 + "1" // "51"

Booleans

false: false, 0, the empty string "", NaN, null, and undefined all become false.
true: everything else.

Comparisons:

<, >, <=, >=, ==, ===, != 1==true // True
1===true // False
1===1 // True

Undefined

An unititialized value. If you declare a variable without assigning a value to it, it will be Undefined.

Null

A deliberate non-value.
